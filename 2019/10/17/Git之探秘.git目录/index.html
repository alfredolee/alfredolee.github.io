Git(05)探秘.git目录

首先，我们进入到.git目录下面，看一下都有些什么文件

1. HEAD 文件

HEAD文件, 当前分支名

2. config 文件

config 文件, 保存仓库的本地配置。

现在我们通过git config user.name 'tudou'修改这个仓库的 user.name。再次查看 config 文件:

同样，我们在 config 文件下面手动删除[user] name = tudou,利用git config --local --list查看本地配置。

3. refs 文件夹

这一串数字是什么意思呢？其实，就是一次 commit 的指针。想要看看eccc4cc74a957d965f543147c24ae996e6e9d331是个什么类型，可以使用git cat-file -t [id]

4. objects 文件夹

这里我们想查看一下3532fec3047ea545a2be930a508068542ee620的类型，报了错。这里查看文件类型要加上父级文件夹的名字，也就是1a。

想要查看1a3532fec3047ea545a2be930a508068542ee620的内容也可以，使用git cat-file -p [id]

5. commit,tree, blob三个对象之间的关系

到目前为止, git 中的是三种核心对象commit,tree, blob我们都已经了解到了。那么，它们之间的关系是什么样呢？

我们使用git cat-file -p [id]查看一个 commit 对象下的内容：

我们可以看到，commit 下面包含一个 tree 对象，那我们再来看看 tree 对象下的内容：

我们可以看到，tree下面包含一个blob对象。到现在我们应该也就能明白了，它们的关系如下:

  • 一个commit对应一个tree
  • tree对象其实就是一个文件夹，例子中也就是git-learn文件夹
  • blob对象实际上就是一个文件, 里面存放文件的内容
  • 另外，git 中blob和文件名无关，只要文件内容一样，blob 就一样

使用find .git/objects/ -type f我们可以看到所有的对象。三种对象都放在 objects 中，一旦文件内容相同，blob 对象也相同，同时也就达到了共用的关系。
