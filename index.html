Git(06)之分离头指针

1. 创建分离头指针

我们从一个 commit 创建一个 HEAD，但是没有指向任何分支，这就是分离头指针。git 友情提醒我们为分离头指针创建一个分支，以保存 commit。

2. 分离头指针下提交 commit

这时候我们把git.html重命名为index.html。

观察git log之后的第一句，我们发现只有HEAD并没有发现指向任何分支。那么此时，我们直接切换分支会发生什么呢？

3. 切换分支到 master

观察历史，发现并没有发现git.html to index.html这个 commit。

4. 所谓分离头指针新创建一个分支

那么，现在我们按照 git 的提醒，新创建一个分支:

观察历史，我们发现已经有了git.html to index.html这个 commit。

5. 总结

分离头指针，存在两面性：

  • 好的一面是，我们可以随便写一些临时代码，当我们有紧急任务的时候，我们可以不用管它，直接切回工作分支
  • 弊端就是, 如果分离头指针的内容你是需要的，你就必须为它创建一个新分支，否则在你切换回分支之后，git 在某个时间点回把分离头指针的内容全部删掉

Git(05)探秘.git目录

首先，我们进入到.git目录下面，看一下都有些什么文件

1. HEAD 文件

HEAD文件, 当前分支名

2. config 文件

config 文件, 保存仓库的本地配置。

现在我们通过git config user.name 'tudou'修改这个仓库的 user.name。再次查看 config 文件:

同样，我们在 config 文件下面手动删除[user] name = tudou,利用git config --local --list查看本地配置。

3. refs 文件夹

这一串数字是什么意思呢？其实，就是一次 commit 的指针。想要看看eccc4cc74a957d965f543147c24ae996e6e9d331是个什么类型，可以使用git cat-file -t [id]

4. objects 文件夹

这里我们想查看一下3532fec3047ea545a2be930a508068542ee620的类型，报了错。这里查看文件类型要加上父级文件夹的名字，也就是1a。

想要查看1a3532fec3047ea545a2be930a508068542ee620的内容也可以，使用git cat-file -p [id]

5. commit,tree, blob三个对象之间的关系

到目前为止, git 中的是三种核心对象commit,tree, blob我们都已经了解到了。那么，它们之间的关系是什么样呢？

我们使用git cat-file -p [id]查看一个 commit 对象下的内容：

我们可以看到，commit 下面包含一个 tree 对象，那我们再来看看 tree 对象下的内容：

我们可以看到，tree下面包含一个blob对象。到现在我们应该也就能明白了，它们的关系如下:

  • 一个commit对应一个tree
  • tree对象其实就是一个文件夹，例子中也就是git-learn文件夹
  • blob对象实际上就是一个文件, 里面存放文件的内容
  • 另外，git 中blob和文件名无关，只要文件内容一样，blob 就一样

使用find .git/objects/ -type f我们可以看到所有的对象。三种对象都放在 objects 中，一旦文件内容相同，blob 对象也相同，同时也就达到了共用的关系。

Git(03)之查看版本历史

之前的两小节我们分别完成了两次提交，那么我们现在怎么查看之前的提交历史呢？git 给我们提供了git log命令：

1. git log

2. git log –oneline

2. git log –oneline

3. git log –all

-all, 是指查看所有分支的 commit 历史。我们使用git checkout -b temp a4e0c87基于第一次提交新建一个分支。此时，使用git log --all。

当然有时候我们不想看到所有的 commit, 我们可以使用git log -n1来查看最近一次 commit。

我们可以看到，在 temp 分支下，我们也看到了 master 分支下的 commit。但是感觉不够明了。

4. git log –all –graph

--graph, 相比之前，添加了一些图形化的标识使得表现更明了。

我们在temp分支，做出一个修改，形成一个 commit。在此 commit 的基础上，新建temp2分支。

我们可以看到，最左侧多了一个图形线，并且master分支是落后temp和temp2分支一个 commit, 上面图标很好的表现出来了。

5. 总结

  • git log –oneline 查看单行的简洁历史。
  • git log –all 查看所有分支的历史
  • git log –all –graph 查看图形化的 log 地址
  • git log –oneline -n4 查看最近的四条简洁历史。
  • git log –oneline –all -n4 –graph 查看所有分支最近 4 条单行的图形化历史。
  • git help –web log 跳转到 git log 的帮助文档网页
